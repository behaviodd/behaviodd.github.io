# 퇴근 계산기 (Work-hours Survival Calculator)

> 남은 근무시간을 나눠서, 오늘 몇 시에 집 갈 수 있는지 계산해주는 모바일 퍼스트 웹앱.

flex · 시프티 · 각종 근태관리 시스템에서 확인한 **남은 인정근무시간**을 기준으로,
남은 근무일 동안 하루에 얼마나 일해야 하는지를 바로 알려줍니다.

- “41시간 54분 남았고 5일 남았는데 하루 몇 시간 일해야 해?”
- “오늘 8시 27분 출근했는데 몇 시에 집 갈 수 있어?”
- “오늘 18시에 퇴근하면 나머지 4일은 몇 시간씩 해야 해?”
- “마지막 3일은 9시간만 체류하고 싶은데 앞 이틀은 얼마나 있어야 해?”

같은 질문에 10초 안에 답하는 걸 목표로 만들었습니다.

---

## 앱 목적

지친 직장인이 밤에 봐도 부담 없이, **가장 중요한 한 문장**(예: “오늘은 17:50에 집
갈 수 있어요.”)을 가장 크게 보여주는 것이 목표입니다. 단순 숫자 계산기가 아니라
실제로 궁금한 시나리오에 바로 답합니다.

## 인정근무시간 vs 체류시간

| 구분 | 의미 | 예시 |
| --- | --- | --- |
| **인정근무시간** | 근태 시스템에 실제로 쌓이는 시간 (보통 휴게 제외) | 8시간 23분 |
| **체류시간** | 출근부터 퇴근까지 회사에 머문 전체 시간 | 9시간 23분 |
| **휴게시간** | 체류에서 빠지는 시간 | 1시간 |

```
체류시간   = 인정근무시간 + 휴게시간
인정근무시간 = 체류시간   − 휴게시간
```

### 휴게시간 기본값

- 기본값은 **1시간(60분)** 입니다.
- 30분 / 45분 / 60분 / 90분 프리셋으로 바로 바꿀 수 있습니다.

## 계산 방식

모든 계산은 **분 단위**로 처리하고, 화면에 보여줄 때만 `시간/분/초` 또는 `HH:MM`
으로 변환합니다.

1. **하루 평균 인정근무** = 남은 인정근무시간 ÷ 남은 근무일 (소수점 그대로 표시)
2. **안전 인정근무** = 위 값을 1분 단위로 **올림** (`Math.ceil`)
3. **하루 체류시간** = 안전 인정근무 + 휴게시간
4. **퇴근 시각** = 출근 시각 + 안전 인정근무 + 휴게시간 (1분 버퍼 옵션)

예시 (남은 41시간 54분 ÷ 5일, 휴게 1시간):

```
하루 평균 8시간 22분 48초  →  안전 8시간 23분 인정근무  →  체류 9시간 23분
출근 08:27 →  퇴근 17:50 (버퍼 적용 시 17:51)
```

### 지원 시나리오

- **A. 기본 하루 목표** — 남은 시간 / 남은 일수 / 휴게
- **B. 출근 기준 퇴근 시각** — 출근 시간을 넣으면 자동 계산
- **C. 오늘 특정 시각 퇴근 시 나머지 날** (예: 오늘 18:00 퇴근)
- **D. 오늘만 N시간 체류 시 나머지 날** (예: 오늘 8/9/9.5시간 체류)
- **E. 며칠만 짧게 체류하는 분산 플랜** (예: 이틀만 9시간 체류)
- **F. 마지막 N일 체류 고정 시 앞날 계산** (예: 마지막 3일 9시간 체류)

### 경고 표시

색만으로 정보를 주지 않도록 항상 텍스트를 함께 보여줍니다.

- 하루 인정근무 **10시간 이상** → 주황 경고
- 하루 체류 **11시간 이상** → 빨강 강한 경고
- 8시간 이하 → 초록(여유)

### 예외 처리

- 남은 일수 0 이하 → 계산 대신 안내
- 남은 시간 0 이하 → “이미 채웠어요”
- 체류 < 휴게 → 인정근무 0 처리 / 오류 안내
- 퇴근 < 출근 → 다음날 계산 미지원, 오류 안내
- 분 입력은 0~59만 허용

## 로컬 실행 방법

> Node.js 18+ 필요 (개발 환경은 Node 22에서 확인).

```bash
cd work-hours-calculator
npm install        # 의존성 설치

npm run dev        # 개발 서버 (http://localhost:5173)
npm test           # 요구사항 테스트 케이스 검증 (vitest)
npm run build      # 배포본 빌드 → 저장소 루트의 /work-hours/ 로 출력
npm run preview    # 빌드 결과 미리보기
```

### 배포 구조 (GitHub Pages)

이 저장소는 Jekyll 기반 GitHub Pages 사이트입니다.

- React 소스: `work-hours-calculator/` (Jekyll 빌드에서 제외)
- 빌드 산출물: 저장소 루트 `work-hours/` → `https://behaviodd.github.io/work-hours/` 로 서빙

`vite.config.ts` 의 `base: "./"` 로 어떤 하위 경로에 올려도 에셋이 깨지지 않습니다.
코드를 수정한 뒤에는 `npm run build` 로 `work-hours/` 를 다시 생성해 함께 커밋하세요.

## 기술 스택

- React 18 + TypeScript
- Vite
- 상태관리는 `useState` 만 사용 (전역 상태 라이브러리 없음)
- 일반 CSS (모바일 퍼스트, 최대 너비 480px)
